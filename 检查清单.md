# QY-RAG iframe 页面设计检查清单

目标：检查除 `/` 根目录外，可能作为 iframe 嵌入使用的页面是否符合 Gemini 风格原则：低视觉负载、靠色差建立纵深、合理圆角、克制阴影、均衡卡片重量、可独立嵌入。

## 检查标准

- 页面根容器使用 `bg-app-page`，嵌入 iframe 时边缘不突兀。
- 页面不依赖外部导航或根布局才能理解当前内容。
- iframe 内不能依赖浏览器后退完成主要返回；详情页需要有清晰的面包屑、返回入口或本页内导航闭环。
- 视觉验收以用户正在观察的真实浏览器窗口为准，不使用固定 viewport 推导设计结论。
- 若页面出现不自适应、右侧空白、错位等异常，先恢复正常显示，再继续优化。
- 有 sidebar 的页面使用 L0/L1/L2 三层纵深：页面底色、主内容、辅助面板。
- 卡片不靠边框建立边界，默认无阴影或极弱阴影，hover/focus/selected 才通过轻阴影表达激活。
- 阴影不作为静态装饰直接表达层级；优先绑定 hover、focus、open、active、selected、dragging 等状态，与轻微动效一起形成反馈。
- 空态、表格、分页、筛选栏在全屏预览下保持原有尺寸，不因视觉优化改变布局。
- 字体尺寸与容器重量匹配，避免大面积空白中孤立小元素。
- 操作按钮集中在 header/tool bar，不独立漂浮。
- 不使用硬算高度，优先 `flex-1 min-h-0`。

## 已发现问题

- 列表页按全屏预览检查尺寸与布局；iframe 只检查页面内返回、上下文与独立嵌入体验。
- `Agents` 创建菜单曾存在按钮嵌套按钮，影响焦点与点击热区。
- 空态卡片原先使用虚线边框和小圆角，作为 iframe 独立页面时视觉较硬。
- `Button` / `Input` / `Select` / `Popover` 共享组件存在偏硬边框风格，与 Gemini 色差分层原则不一致。
- `Search` 设置面板使用硬算高度与 absolute 底部操作区，iframe 内嵌时容易与容器高度脱节。
- Chunk 解析面板残留旧 token、硬左边框和 `vh` 高度，视觉上和详情页 surface 体系不一致。
- `bg-app-page` 曾被固定成浅色，导致 dark 模式下页面大背景仍然发亮。

## 已调整

- 统一 `bg-app-page` 为 Gemini 蓝调浅灰 `#E8EDF5`。
- 空态卡片改为 `bg-bg-base shadow-none`，弱化硬边框。
- 共享按钮、输入框、下拉、弹层减少硬边框，改用透明边界、色差和 focus ring；表格尺寸骨架保持原样。
- 列表页空态根容器补齐 `bg-app-page`。
- `Agents` 创建按钮改为 `DropdownMenuTrigger asChild`，消除按钮嵌套。
- `Dialog` / `AlertDialog` / `DropdownMenu` / `Command` / `Tooltip` / `HoverCard` 统一为轻遮罩、轻阴影和项目 text/surface token。
- `Search` 首页和结果页弱化硬边框，摘要、metadata、来源文档和分隔使用 `bg-bg-card` 色差表达。
- `SearchSetting` 改为 flex 流：内容区滚动，底部操作区回到正常布局，移除硬算高度。
- `Agent` 详情外层补齐 `bg-app-page` 与 flex 容器，作为 iframe 独立页时边缘一致。
- Chunk 主面板移除旧色彩 token、硬左边框和 `vh` 高度，卡片回到 `bg-bg-card`。
- `bg-app-page` 改回 CSS 变量，并在全局定义 light `#e8edf5` / dark `#131314`，恢复 dark 模式 L0 页面底色。
- 新增 `shadow-surface` / `shadow-raised` / `shadow-floating` / `shadow-modal` / `shadow-focus` token，并把按钮、输入框、卡片、下拉、弹层统一为激活态阴影反馈。
