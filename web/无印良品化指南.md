## 五、翻译文件中的 RAGFlow（低优先级）

翻译文件（`src/locales/*.ts`）中存在大量在 tooltip / 功能说明中引用 "RAGFlow" 的文字，涉及 16 个语言文件、约 300+ 处。这些文字仅在 hover 提示和配置说明中展示，不在主界面直接可见。

**处理原则**：
- 主界面直接可见的 → 必须改
- Tooltip / 说明性文字 → 按需替换为品牌中性表达（如"系统"、"本平台"）或直接删除该句

**批量替换建议**：
```bash
# 仅替换独立出现的品牌词，避免误伤技术名词
sed -i 's/RAGFlow/系统/g' src/locales/zh.ts
sed -i 's/RAGFlow/the system/g' src/locales/en.ts
# 其余语言同理
```

## 六、iframe 嵌入模式下的额外检查

嵌入第三方系统时，以下位置可能额外暴露品牌信息：

- **分享页** `next-chats/share`、`agent/share`、`next-search/share` — 检查页面标题和 Logo
- **Widget 页** `next-chats/widget` — 检查 `<title>` 和加载动画
- **Embed Dialog** 生成的嵌入代码中的 `src` 域名 — 如需隐藏源站，由后端处理

---

## 七、检查工具命令

每次交付前，用以下命令扫描残留：

```bash
# 扫描用户可见的品牌文字（JSX 文本节点和 HTML 属性）
grep -r '"RAGFlow\|>RAGFlow\|title.*RAGFlow\|alt.*RAGFlow' src/ --include="*.tsx" --include="*.html"

# 扫描外部链接
grep -r 'ragflow\.io\|infiniflow\|discord\.com\|github\.com/infiniflow' src/ --include="*.tsx"

# 扫描 logo 图片引用
grep -r 'logo\.svg' src/ --include="*.tsx"
```
